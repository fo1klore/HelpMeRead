# 课程设计指南

本文件指导板块 B 课程化的**内容设计**。所有课程用 **Markdown（使用 Obsidian 兼容语法：callout、MathJax、`[[双链]]`、`<details>` 折叠）** 产出。引用的 frontmatter 字段见 `references/frontmatter-schema.md`。

所有课程共用：**弹性节数**、**概念粒度**、**单课模块**、**新手友好 + 可跳过标注**。差别在组织逻辑（按文献类型分三种）。

---

## 共通规则

### 节数控制（弹性）

- **默认 3-5 节**，这是推荐范围
- **最少 1 节**——短文/短论点文不凑数，1-2 节能讲清就 1-2 节
- **超过 7 节时**必须在大纲阶段向用户说明理由（如"这篇综述覆盖 X 个独立流派，建议拆成 8 节"）
- **综述单独放宽到 6-8 节**，每个主要流派独立一节，小众流派在全景图中提及但不独立成节
- 篇幅参考：短论文（< 8 页）2-3 节；中等篇幅 3-4 节；长论文 4-6 节；大综述 6-8 节
- **不再有"宁可单节内容稍多也不要突破上限"的硬约束**

### 文件与目录

目录和文件命名遵循 `references/frontmatter-schema.md` 中的命名约定。

### 节角色（Core / Supporting）

每节在课程大纲阶段必须声明角色。角色决定该节适用的深度标准和自测要求，在"生成后完整性检查"（SKILL.md step 2.6）中验证。

- **Core**：承载实质性学习负载的节（概念讲解、公式推导、算法分析、机制拆解等）。本节至少包含一个核心概念或公式。Core 节必须声明学习负载（本节要讲什么）、适用的深度标准，以及至少 1 道覆盖本节核心内容的自测题。
- **Supporting**：提供背景、动机、上下文铺垫或过渡连接的节。不引入新核心概念。自测可降级为 0-1 题（无需覆盖核心概念）。

**课程级约束**：Core 节数 ≥ max(ceil(总节数 × 50%), 1)。若全文只有 1 节，该节必须为 Core。不满足时触发 step 2.6 重试。

> 判断标准：一节能否在不引用"本章未讲解的前置概念"的情况下，让零基础读者获得一个可独立陈述的知识点（能回答"这节课我学到了什么"且答案非"了解了背景"）。能→Core；不能→Supporting。

### 单课模块（顺序固定）

每节课文件严格按以下顺序组装。各模块详细格式规范见后续对应节。

**前置（每节必有，标题前）：**
0. **课程地图**（文件最开头）→ 详见"课程地图"节
1. **可跳过标注**（条件性，地图之后、学习目标之前）→ 详见"可跳过标注"节
   - 超短概念/术语索引（可选，概念 ≥ 5 时）：放在可跳过标注之后、学习目标之前

**正文模块（必有模块缺一不可，可选模块按条件出现）：**
2. **概念/术语首次出现解释**（分散嵌入正文，不设独立模块）→ 详见"概念/术语分散嵌入规则"节
3. **学习目标 + 动机**
4. **逐概念教学循环**（重复：每节内每个核心概念各自走完完整教学链条，不合并讲解。详见"模块 4"）
5. **(可选) 💻 代码实现**（紧跟在对应概念的教学循环之后，随概念重复，不批处理）
6. **自测小问题**
7. **小结 + 预告**
8. **(可选) 🔗 延伸**

**结尾（每节必有）：**
9. **(仅末节) 🎓 下一步拆解提醒** → 详见"末节收尾提醒"节
10. **底部导航**（文件末尾）→ 详见"底部导航"节

### 概念/术语分散嵌入规则

不设独立热身模块。任何需要解释的概念或术语（核心概念、前置术语、背景术语），在其**正文首次出现处**立即解释，不先集中列出。三者使用统一的分散嵌入策略：

- **核心概念**：首次出现时用 `[[概念名]]` 双链包裹，并紧跟一句自然语言解释。格式：`句子开头 ... [[self-attention]]（自注意力机制，让每个词关注所有其他词来理解上下文）... 句子继续`。若结构较复杂，可在首次出现句后用行内 callout 展开说明。
- **前置术语**：首次出现时用加粗标记并紧跟一句解释，**不**用 `[[]]` 包裹。格式：`这里涉及 **KL 散度**（Kullback-Leibler divergence，衡量两个概率分布差异的指标）`。
- **背景术语**：首次出现时一句话自然语言解释。格式：`$\sigma(\cdot)$ 是 sigmoid 函数，将输入压缩到 (0,1) 区间`。

**统一约束**：
- 解释句必须用日常语言，不得包含尚未在本节解释的领域术语
- 同一术语再次出现不再重复解释
- 若某概念依赖另一前置概念，被依赖概念必须在正文中先出现并先解释
- 跨节概念依赖：概念 A（第 3 节）依赖概念 B（第 5 节）→ B 在第 5 节首次出现时标注导航"详见第 × 节"
- **超短概念/术语索引**（可选，非强制）：若本节涉及概念/术语较多（≥5 个），放在可跳过标注之后、学习目标之前，列概念名 + `→ §X.Y` 导航。格式示例：

  ```
  > [!abstract]- 📑 本节涉及概念
  > [[self-attention]] · [[multi-head attention]] · position encoding
  ```

  索引不附定义——定义在正文首次出现处提供。索引不是热身模块的替代，仅作导航辅助，单节概念数 <5 时省略。

### 模块 3：学习目标 + 动机
   - 本节学什么、为什么需要它
   - 承接上文："上一节我们知道了 X，这一节解决 Y"
   - 用 `> [!objective]+ OBJECTIVES` callout（**默认展开**）

### 模块 4：逐概念教学循环（每个核心概念重复）

一节可以包含多个核心概念（如"自注意力机制"节同时讲解 Q/K/V、attention scores、softmax 归一化）。**不**先将所有概念的类比集中在一起、再把所有公式集中在一起——而是每个概念各自走完完整的教学链条，一个讲完再讲下一个。对**每个**核心概念依次执行：

**结构模板**（每个概念独立走一遍）：
```
[概念名] 直觉类比（> [!analogy]）
→ 形式化定义 + 公式逐步推导
→ 数值例子 / 示例
→ 嵌入原图（如有）
→ (可选) 💻 代码实现 —— 紧跟在概念后
```

**细节规则**（以下规则适用于每个概念的独立教学循环）：
   - 先用日常类比讲直觉，再给形式化定义
   - 类比用 `> [!analogy]+ ANALOGY` callout（**默认展开**，暖色背景）
   - 公式用 MathJax（`$...$` 行内、`$$...$$` 块级，Obsidian 原生渲染），配逐步推导
   - 每个新核心概念首次出现时加粗并一句话解释。**解释句不得包含尚未在本节解释的领域术语**——例如 `Transformer 使用了 self-attention` 不算合格解释（self-attention 是嵌套术语），而 `Transformer 是一种基于自注意力的神经网络，它通过 attention 机制让每个词关注所有其他词来理解上下文` 是合格解释（即使读者不知道 attention，这句话也给予了基本认知）
   - 核心概念**首次出现时用 `[[概念名]]` 双链包裹**，作为原子笔记候选标记。只有满足"原文给出定义/公式推导 + 在论文贡献中有地位"的概念才标注双链
   - 关键结论和实验数据标注原文出处——在正文后另起一行，与正文间隔一个空行，前缀 `📖`。格式示例：
     ```markdown
     📖 第 12 页 / Sec.3.2, Figure 2
     ```
   - **断言语义标注**：所有非原文信息必须标注以下标签之一，标注位置为段落或句子末尾的方括号内：
     - `[背景]`：通用领域背景知识，不来自本篇论文（如"Transformer 是 2017 年提出的"——这是领域常识，不是本篇论文的观点）
     - `[推断]`：基于论文内容的模型延伸思考、类比推论、后续工作链接、成本估计等非原文内容（如"DPO 的训练流程和 SFT 非常接近，只需要修改 loss 计算部分"——这是模型对两者关系的理解，不是论文原文）
     - **铁律**：没有原文或外部来源支持的强断言（如"DPO 成本是 PPO 的 1/3 到 1/2"）必须在末尾标注 `[推断]`。未标注此类标签的强断言视为不通过完整性检查
     - 已标注 `📖` 来源的正文视为隐含 `[原文]`，可省略标签
     - 标注位置示例：
       ```markdown
       这一结果表明，在 IMDB 情感分类任务上，DPO 的准确率比 PPO 高约 2 个百分点。 [推断]
       📖 Sec 5.1, Table 2
       ```
   - arxiv 论文的页码标签**做成可点击链接**（链到 arxiv HTML 对应锚点），非 arxiv 保持纯文字
   - **解释深度**：目标是让学习者达到**精读一篇论文**的理解水平，而非泛读。每个核心概念至少覆盖：
     (1) 一句直觉类比，让零基础也能大致理解；
     (2) 完整的形式化定义/公式，逐一拆解每个参数、变量的含义和取值范围；
     (3) 这篇论文为什么这样设计（motivation）——和其他方法的对比、trade-off 分析、设计取舍的原因；
     (4) 这个概念在整篇论文中的位置——它解决了前文的哪个问题、为后文的哪个结论铺路。
     用比原文更紧凑直白的语言，但信息量不低于原文
   - **教学深度最低结构**（Core 节强制，推导类节必须全部满足、非推导类 Core 节适用其中 3 条）：
     (1) **问题动机**：为什么需要这个公式/方法，它解决前文的什么问题
     (2) **变量准备**：列出所有符号含义（变量名、取值范围、物理/语义含义）
     (3) **分步推导**：要求拆成至少 3 个小步骤，每步配一句"这一步在做什么"的自然语言说明
     (4) **直觉解释**：推导完成后，用自然语言回述"这个公式/方法本质上在做什么"
     (5) **数值例子**：取一组小整数/简单向量代入公式，逐步演算并写出中间结果
     (6) **错误直觉**：指出至少一个常见误解或初学者的典型错误推理
     非推导类 Core 节（纯概念解释、机制分析、架构讲解）适用 (1)(2)(4)(5)（(2)按需），(3)(6)不强制
   - **示例嵌入**：每个核心概念讲解后，若满足以下条件之一则自动嵌入贴合示例：
     - 有数学公式 → 设计示例数据（小整数/简单向量），代入公式逐步演算，写出中间结果
     - 有算法步骤/流程 → 用具体输入值完整走一遍流程，每个步骤写出当前状态
     - 有架构图/流程图 → 用具体数据从入口跟踪到出口，展示各阶段形态变化
     - 有复杂度分析 → 取具体数值 $n$ 代入 $O()$ 公式，对比不同规模下的运算量
     - 有对比/权衡 → 对同一问题用两种方案分别实现，展示输出差异
     - 有抽象概念/定义 → 先用日常类比建立直觉，再立即映射回技术语境
     示例紧跟在对应概念解释之后，用自然段落或代码块呈现（不加额外 callout）。
     **来源标注**：原文自带示例直接展示；agent 构造示例末尾标注 `*✏️ 构造示例*`。
     **约束**：每个概念最多 1 个示例；数值演算 ≤ 100 字（不含公式），代码 ≤ 15 行；构造不了就跳过，不硬给。
   - **嵌入原图**：当论文有架构图/流程图/关键结果图时，把图抓到 `assets/` 并用 `![[assets/figure-N.png]]` 嵌入对应讲解段落后。每张图标注原 Figure 编号。这是最高教学价值的内容，不要遗漏
   - **图片质量控制**：只提取 PDF **内嵌图对象**（PyMuPDF `page.get_images()` / pdfimages），**禁用整页栅格化**（`page.get_pixmap()` / pdftoppm）——后者得到的是整页论文截图而非单张图。提取后丢弃长或宽 < 100px 的噪音图（图标/装饰元素）；只把确定要嵌入的图存入 `assets/`，课程 + 笔记全部生成完毕后扫描删除未被 `![[...]]` 引用的残留图。详见 SKILL.md 步骤 1 图片抽取
   - **⚠️ 铁律**：所有图必须从原文实际提取，**严禁用 ASCII art、文字描述或自行绘制的示意图替代原文图片**。若某图确实无法提取，在对应位置标注「📎 参见原文 Figure N」，不自行描绘。

### 模块 5：(可选) 💻 代码实现（跟概念走，非独立成节）

本模块不是独立的一节课阶段——它紧跟在每个概念的模块 4 教学循环之后，随概念重复。
   - **触发条件**：该概念在论文中对应可独立实现的函数/类/方法，且存在官方仓库或公认社区实现
   - **获取方式**：用 WebFetch 抓取 GitHub raw 文件，**不凭记忆编造**；每个概念选取对应的一段（通常 5-25 行）；标注来源（仓库名 + 文件路径 + 大致行号）
   - **展示格式**：\`\`\`语言名 代码块 + 逐行注释（`# 对应公式 1 的 QK^T 部分`），说明每行对应论文中的哪个公式或哪个设计决策
   - **出现位置**：紧跟在对应概念的讲解段落之后，与概念绑定而非独立成节
   - **原则**：代码片段实时抓取，抓取失败或代码混杂过多工程细节时省略该段；同一节课可多次出现

### 模块 6：自测小问题
   - 简单节：1 题；重难点：2-3 题
   - 用 `> [!quiz] CHECK YOURSELF` callout（**展开**，问题直接可见），答案用嵌套 `> [!answer]- 答案` callout（**折叠**，只折答案不折问题）。结构如下：
     ```markdown
     > [!quiz] CHECK YOURSELF
     > **问题：** 为什么要除以 $\sqrt{d_k}$？去掉它会怎样？
     >
     > > [!answer]- 答案
     > > 当 $d_k$ 较大时，Q 和 K 的点积值变得很大（方差 = $d_k$），把 Softmax 推到饱和区导致梯度接近 0。除以 $\sqrt{d_k}$ 将方差拉回 1，保持梯度健康。
     ```
   - 题型以简答为主（"解释为什么""说明关系""对比异同""用自己的话描述"），不出现选择题和判断题
   - **内容范围约束**：所有自测题的答案必须在本节核心讲解中有直接对应段落。不得考察本节未讲解的概念、未推导的公式或未展示的数据。即"讲什么考什么"原则。
     - 若一道题的答案需要用到本节课未讲解的知识，则该题不合格
     - 简答类题型天然符合此原则——解释/对比/用自己话说，都是对已学内容的巩固

### 模块 7：小结 + 预告
   - 用 `> [!summary]+ SUMMARY` callout（**默认展开**——小结是用户学完一节的核心收获，折叠易被跳过）
   - 3-5 个要点 bullet
   - 1 句话预告下节

### 模块 8：(可选) 🔗 延伸
   - 当有公认高质量的外部资源（3Blue1Brown、distill.pub、Jay Alammar、经典 lecture 视频、官方文档等）时追加
   - 用 `> [!link]+ 延伸资源` callout（**默认展开**——精选资源是核心收获，折叠易被跳过）
   - 精选 1-3 个链接，**链出去不抓内容**，附一句话简介
   - **原则**：宁缺毋滥。只在确有高质量资源时才出现，不做搜索引擎的搬运工
   - 与"待学习"机制的区别：延伸资源是精选的、主动给的；待学习是用户被概念卡住时按需触发的

### 底部导航（结构，不计入模块）

每节课文件最后一个结构性元素必须是上下节导航链接，格式：

```
← [[上一节slug|第 N 节 · 标题]] ｜ [[下一节slug|第 N 节 · 标题]] →
```

- **首节**（N=1）：左侧显示 `← 无`
- **末节**（N=总节数）：右侧显示 `课程结束`
- **中间节**：左右均为完整链接
- 导航前用 `---` 分隔线与上方内容隔开
- **不可省略**，每节课文件必须包含

### 末节收尾提醒（仅最后一节）

最后一节的 SUMMARY 之后、底部导航之前，追加一个**独立的醒目 callout**，明确提示学完后会进入拆解笔记阶段（其余节不加）：

```
> [!tip]+ 🎓 下一步：拆解原子笔记
> 你已完成全部课程！接下来我会引导你为笔记里的**核心概念**逐一写下自己的理解，用自己的话写出定义、加深理解。准备好后告诉我即可开始。
```

> 放在 SUMMARY 外部、独立成块：与课程小结区分开，作为"学习阶段→巩固阶段"的交接提示，默认展开确保用户看到。

### 课程地图

每节课文件**最开头**（在可跳过标注之前）放一个折叠的课程全景地图，列出全部课程节标题，当前节加 `→` 标记 + 加粗。用户一眼看到自己在整个课程中的位置。

格式：
```
> [!map]- 📍 课程地图（共 N 节）
> 1. 为什么需要 Attention
> **→ 2. 自注意力机制**
> 3. 多头注意力与位置编码
> 4. 实验结果与影响
```

每节课都必须有，不可省略。

### 可跳过标注

标注位置必须固定，不可按内容逻辑随意放置。

**模板**（紧跟在每节的课程地图之后、学习目标之前）：
```
<!-- 可跳过标注：固定在此位置，不得挪动 -->
```

## 类型 1：研究论文 —— 学习路径重组

**核心思想**：不拘泥原文顺序，按"学习者从零到懂"的认知曲线重组。

**典型路径**（弹性节数）：
1. **背景与直觉**：这个领域在解决什么问题、为什么难（铺背景）
2. **核心方法**：本文最关键的 1 个创新点（讲透）
3. **方法细节**（可选）：第 2 节方法的进阶部分或第二个创新点
4. **实验与结论**：效果如何、有什么启示

**示例**（以 Attention Is All You Need 为例，4 节）：
- 01-why-attention.md（RNN 的瓶颈 → 为什么需要 attention）
- 02-self-attention.md（核心机制 + 公式 + 论文原图 Figure 1）
- 03-multi-head-and-position.md（多头注意力 + 位置编码）
- 04-results-and-impact.md（翻译任务结果、对比 Table 1、影响与局限）

## 类型 2：综述 —— 全景 + 流派逐节

**核心思想**：综述的价值是"地图"，课程要帮学习者建立鸟瞰图。

**典型路径**（6-8 节，每个主要流派独立一节）：
1. **全景地图**：这个领域怎么分类、各流派大致位置（必须第一节）
2. **流派 A**：某一类方法的代表思想、优劣 + 关键技术细节
3. **流派 B**：另一类方法
4. **流派 C**：第三类
5. **流派 D**（可选）：第四类，或趋势/开放问题
6. **全景回顾 + 趋势**（可选）：收束，展望

**注意事项**：
- 综述涉及的流派多，**每个主要流派独立一节**，不做硬合并
- 小众流派在全景地图中简单提及即可，不独立成节
- 第一节全景图要让学习者离开时能在脑子里画出"这个领域有哪些路"
- 每个流派节用统一结构：核心主张 → 代表方法 → 关键公式/架构 → 优势 → 局限

## 类型 3：数据集论文 —— 数据生命周期

**核心思想**：按数据从无到有的阶段组织，让学习者理解一个数据集是怎么造出来的。

**典型路径**（弹性节数）：
1. **全景**：这个数据集是什么、为什么重要、填补什么空白
2. **采集**：数据从哪里来、怎么收集
3. **标注**：标注方案与质量控制
4. **评测应用**（可选）：基于这个数据集能做什么任务、已有结果

**注意事项**：
- 统计与偏差分析融入对应阶段的讲解，不单独成节（避免枯燥数字堆砌）
- **大表格处理**：数据集论文常有大段统计表（规模/分布/评分）。**不**把全表搬入课程——摘选关键代表性数据（前 5-10 项 + 总计/均值），用紧凑要点列表，末尾注明"完整表格见原文【第 X 页 / 表 Y】"
- 如果数据集有明显的伦理争议，在最后一节单独讲
