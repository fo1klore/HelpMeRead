# 课程设计指南

本文件指导板块 B 课程化的**内容设计**。HTML 外壳模板见 `course-template.html`。

所有课程共用：**3-5 节**、**概念粒度**、**单课五模块**、**新手友好 + 可跳过标注**。差别在组织逻辑（按文献类型分三种）。

---

## 共通规则

### 节数控制
- 硬性范围：**3-5 节**
- 短论文（< 8 页）取 3 节；中等篇幅 4 节；长论文或综述 5 节
- 宁可单节内容稍多，也不要突破 5 节上限

### 单课五模块（顺序固定）

1. **学习目标 + 动机**（约 50 字）
   - 本节学什么、为什么需要它
   - 承接上文："上一节我们知道了 X，这一节解决 Y"

2. **核心讲解 + 类比**
   - 先用日常类比讲直觉，再给形式化定义
   - 公式用 KaTeX 渲染（`$...$` 行内、`$$...$$` 块级），配逐步推导
   - 每个新术语首次出现时加粗并一句话解释
   - 关键结论和实验数据标注原文出处【第 N 页 / Sec.X / Table Y】，不逐句标注
   - **解释深度**：目标是让学习者达到**精读一篇论文**的理解水平，而非泛读。每个核心概念至少覆盖：(1) 一句直觉类比，让零基础也能大致理解；(2) 完整的形式化定义/公式，逐一拆解每个参数、变量的含义和取值范围；(3) 这篇论文为什么这样设计（motivation）——和其他方法的对比、trade-off 分析、设计取舍的原因；(4) 这个概念在整篇论文中的位置——它解决了前文的哪个问题、为后文的哪个结论铺路。用比原文更紧凑直白的语言，但信息量不低于原文

3. **(可选) 💻 代码实现**（按概念追加，可多次出现）
   - **触发条件**：该概念在论文中对应可独立实现的函数/类/方法，且存在官方仓库或公认社区实现
   - **获取方式**：用 WebFetch 抓取 GitHub raw 文件，**不凭记忆编造**；每个概念选取对应的一段（通常 5-25 行）；标注来源（仓库名 + 文件路径 + 大致行号）
   - **展示格式**：代码块 + 逐行注释，说明每行对应论文中的哪个公式或哪个设计决策
   - **出现位置**：紧跟在对应概念的讲解段落之后，与概念绑定而非独立成节
   - **原则**：代码片段实时抓取，抓取失败或代码混杂过多工程细节时省略该段；同一节课可多次出现

4. **自测小问题**（按难度浮动）
   - 简单节：1 题
   - 重难点：2-3 题
   - 采用翻转卡片形式：正面显示"点击卡片翻转查看答案"，背面直接写出答案内容。**答案不加"答案："等前缀**——正面已有明确提示
   - 题型以简答为主（"解释为什么""说明关系""对比异同""用自己的话描述"），不出现选择题和判断题

5. **小结 + 预告**
   - 3-5 个要点 bullet
   - 1 句话预告下节

### 可跳过标注
每节顶部（学习目标之前）放一行：
```
> 可跳过：若你已熟悉 <X 概念>，可跳过本节直接进入第 N 节
```
只在确实有前置知识可跳过时才标注，不是每节都标。

### 文件命名
`01-<概念名>.html`、`02-<概念名>.html`……存到 `courses/<文献简称>/`。概念名用英文或拼音短词，避免空格和特殊字符。

### 表格展示
遇到论文中的关键结果表格（如模型对比表、消融分析表），先用 HTML `<table>` 完整展示，再紧跟一段文字总结这张表最关键的结论。不先总结后引用，也不只摘选数据。

## 类型 1：研究论文 —— 学习路径重组

**核心思想**：不拘泥原文顺序，按"学习者从零到懂"的认知曲线重组。

**典型路径**（3-5 节）：
1. **背景与直觉**：这个领域在解决什么问题、为什么难（铺背景）
2. **核心方法**：本文最关键的 1 个创新点（讲透）
3. **方法细节**（可选）：第 2 节方法的进阶部分或第二个创新点
4. **实验与结论**：效果如何、有什么启示

**示例**（以 Attention Is All You Need 为例）：
- 01-why-attention.html（RNN 的瓶颈 → 为什么需要 attention）
- 02-self-attention.html（核心机制 + 公式）
- 03-multi-head.html（多头注意力）
- 04-results.html（翻译任务结果与影响）

---

## 类型 2：综述 —— 全景 + 流派逐节

**核心思想**：综述的价值是"地图"，课程要帮学习者建立鸟瞰图。

**典型路径**（3-5 节）：
1. **全景地图**：这个领域怎么分类、各流派大致位置（必须第一节）
2. **流派 A**：某一类方法的代表思想、优劣
3. **流派 B**：另一类方法
4. **流派 C**（可选）：第三类，或趋势/开放问题

**注意事项**：
- 综述涉及的流派多，但**不超过 5 节**——合并相近流派，只讲代表性的 3-4 个
- **流派合并原则**（按优先级依次应用，直到流派数 ≤ 3-4）：
  1. 技术路线相近的合并（如 encoder-decoder 与 decoder-only 归一类）
  2. 年代迭代关系合并（如"早期统计方法→神经网络方法→预训练方法"归一条演进线）
  3. 影响力较小/小众的合入"其他方向"一节或略去，在全景地图中简单提及即可
- 第一节全景图要让学习者离开时能在脑子里画出"这个领域有哪些路"
- 每个流派节用统一结构：核心主张 → 代表方法 → 优势 → 局限

---

## 类型 3：数据集论文 —— 数据生命周期

**核心思想**：按数据从无到有的阶段组织，让学习者理解一个数据集是怎么造出来的。

**典型路径**（3-5 节）：
1. **全景**：这个数据集是什么、为什么重要、填补什么空白
2. **采集**：数据从哪里来、怎么收集
3. **标注**：标注方案与质量控制
4. **评测应用**（可选）：基于这个数据集能做什么任务、已有结果

**注意事项**：
- 统计与偏差分析融入对应阶段的讲解，不单独成节（避免枯燥数字堆砌）
- **大表格处理**：数据集论文常有大段统计表（规模/分布/评分）。**不**把全表搬入 HTML——摘选关键代表性数据（前 5-10 项 + 总计/均值），转为 HTML 的紧凑要点列表，末尾注明"完整表格见原文【第 X 页 / 表 Y】"
- 如果数据集有明显的伦理争议，在最后一节单独讲

## 课程大纲呈现（学习开始前）

生成全部 HTML 后，向用户展示大纲，格式：

```
📚 <文献标题> 课程大纲（共 N 节）

第 1 节：<标题>
  可跳过：<条件 或 "无">

第 2 节：...

💡 提醒：学习中遇到文献没解释的概念，随时告诉我，我会帮你存到待学习目录并协助找资料。
```

展示后**立即**提醒待学习机制，然后用 `start "" "<第1节路径>.html"` 打开第一节。
