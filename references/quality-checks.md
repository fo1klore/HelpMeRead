# 生成后完整性检查（按需加载）

> SKILL.md 步骤 2 B.6 引用本文件。生成后所有课程文件落盘后执行。

---

## 检查列表（逐节执行）

全部课程文件落盘后，逐节执行以下检查项。任一不通过则立即重新生成该节并替换。重试上限 2 次，2 次仍不通过则保留最后一版并标注 `> [!WARNING] 本节完整性检查未通过`，不阻塞管线。

### 1. 块配对

`$` 个数为偶数、`` ``` `` 个数为偶数、`> [!xxx]` 计数成对。

### 2. 末段有完整句

末段（最后 200 字符内）至少有一个 `.` `。` `!` `！` `?` `？` `；` `…` 后跟非空字符。

### 3. 末字符不在句中

末字符不是 `,` `，` `:` `：` `;` `；` `、` `(` `（` 等中间/起始标点。

### 4. 原子笔记双链校验

扫描 `concepts/*.md` 中所有 `[[...]]` 链接（去除 `|` 后显示文本），检查每个链接目标在 `concepts/` 目录下是否有对应 `.md` 文件。不通过则修正双链文本为映射表中的 `filename_slug`。

### 5. 节角色检查

扫描每节课程文件开头，检查是否存在 `role: Core` 或 `role: Supporting` 声明。统计全课程 Core 节数，验证 Core 节数 ≥ max(ceil(总节数 × 50%), 1)。不满足则判定不通过，触发重试。

### 6. Core 节深度检查

对每个 Core 节，逐条验证：
- (a) **推导分步**：如本节含公式，检查公式前有动机、后有 ≥3 步推导、步间有自然语言说明
- (b) **前置桥接**：被引用的非白名单概念有定义或桥接解释
- (c) **数值示例**：如含形式化定义/公式，其后有具体代入示例
- (d) **错误直觉**：如为推导类 Core 节，包含至少一处常见误解指出

任一 Core 节任一条不满足 → 判定该节不通过，触发重试。

### 7. 概念解释覆盖

扫描每节所有 `[[概念名]]` 双链，逐一检查：
- (a) 首次出现时有一句话自然语言解释（非术语堆砌）
- (b) 解释句本身不含尚未解释的领域术语
- (c) 概念间前置依赖完整

不通过则重新生成该节并替换，**重试上限 2 次**。2 次后仍不通过 → 保留最后一版，标注 `> [!WARNING] 本节概念/术语检查未通过`，继续后续流程。

### 8. 来源标注检查

扫描所有 `📖` 来源标注行，验证：
- (a) 每个标注包含具体定位（章节号/图号/表号/页码）
- (b) 标注对应的图号在提取的图片中能找到
- (c) **生成时来源标注**：公式、变量定义、实验结果等核心内容在生成时标注来源章节号

任一不满足则判定不通过，重试该节。

### 9. 断言标签检查

扫描全节，对以下内容检查是否标注了语义标签：
- (a) 通用背景知识/领域事实 → 标注 `[背景]`
- (b) 模型推断/延伸思考/类比推论等非原文内容 → 标注 `[推断]`
- (c) **铁律**：没有任何原文或外部来源支持的强断言必须在末尾标注 `[推断]`
- (d) 已标注 `📖` 来源的正文视为隐含 `[原文]`，可省略标签

### 10. 首次出现缩写解释检查

扫描本节中所有大写缩写（2-6 个大写字母），检查首次出现时是否附带一句解释（全称 + 基本含义）。对于领域内常见通用缩写（AI、ML、NLP、CPU、GPU），若上下文已清晰则豁免。

## 检测命令

```bash
# 块配对
for f in papers/*/course/*.md; do
  dollars=$(grep -o '\$' "$f" | wc -l)
  fences=$(grep -o '```' "$f" | wc -l)
  echo "$f: dollars=$dollars fences=$fences"
done

# 末字符检查
python3 -c "
import glob
for f in glob.glob('papers/*/course/*.md'):
    with open(f) as fh:
        text = fh.read()
    tail = text[-200:] if len(text) > 200 else text
    last_char = text[-1] if text else ''
    has_complete = any(c in tail for c in ['.', '。', '!', '！', '?', '？', ';', '；', '…'])
    in_middle = last_char in [',', '，', ':', '：', ';', '；', '、', '(', '（']
    print(f'{f}: complete_sent={has_complete} mid_char={in_middle}')
"

# 缩写检查
for f in papers/*/course/*.md; do
  grep -oP '\b[A-Z]{2,6}\b' "$f" | sort -u
done
```
