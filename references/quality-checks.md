# 生成后完整性检查（按需加载）

> SKILL.md 步骤 2 B.6 引用本文件。生成后所有课程文件落盘后执行。

---

## 检查列表（逐节执行）

全部课程文件落盘后，逐节执行以下检查项。任一不通过则立即重新生成该节并替换。重试上限 2 次，2 次仍不通过则保留最后一版并标注 `> [!WARNING] 本节完整性检查未通过`，不阻塞管线。

### 1. 块配对

`$` 个数为偶数、`` ``` `` 个数为偶数、`> [!xxx]` 计数成对。

### 2. 末段有完整句

末段（最后 200 字符内）至少有一个 `.` `。` `!` `！` `?` `？` `；` `…` 后跟非空字符。

### 3. 末字符不在句中

末字符不是 `,` `，` `:` `：` `;` `；` `、` `(` `（` 等中间/起始标点。

### 4. 原子笔记双链校验

扫描 `concepts/*.md` 中所有 `[[...]]` 链接（去除 `|` 后显示文本），检查每个链接目标在 `concepts/` 目录下是否有对应 `.md` 文件。不通过则修正双链文本为映射表中的 `filename_slug`。

### 5. 节角色检查

扫描每节课程文件开头，检查是否存在 `role: Core` 或 `role: Supporting` 声明。统计全课程 Core 节数，验证 Core 节数 ≥ max(ceil(总节数 × 50%), 1)。不满足则判定不通过，触发重试。

### 6. Core 节深度检查

对每个 Core 节，逐条验证：
- (a) **推导分步**：如本节含公式，检查公式前有动机、后有 ≥3 步推导、步间有自然语言说明
- (b) **前置桥接**：被引用的非白名单概念有定义或桥接解释
- (c) **数值示例**：如含形式化定义/公式，其后有具体代入示例
- (d) **错误直觉**：如为推导类 Core 节，包含至少一处常见误解指出

任一 Core 节任一条不满足 → 判定该节不通过，触发重试。

### 7. 概念解释覆盖

扫描每节所有 `[[概念名]]` 双链，逐一检查：
- (a) 首次出现时有一句话自然语言解释（非术语堆砌）
- (b) 解释句本身不含尚未解释的领域术语
- (c) 概念间前置依赖完整

不通过则重新生成该节并替换，**重试上限 2 次**。2 次后仍不通过 → 保留最后一版，标注 `> [!WARNING] 本节概念/术语检查未通过`，继续后续流程。

### 8. 来源标注检查

扫描所有 `📖` 来源标注行，验证：
- (a) 每个标注包含具体定位（章节号/图号/表号/页码）
- (b) 标注对应的图号在提取的图片中能找到
- (c) **生成时来源标注**：公式、变量定义、实验结果等核心内容在生成时标注来源章节号

任一不满足则判定不通过，重试该节。

### 9. 断言标签检查

扫描全节，对以下内容检查是否标注了语义标签：
- (a) 通用背景知识/领域事实 → 标注 `[背景]`
- (b) 模型推断/延伸思考/类比推论等非原文内容 → 标注 `[推断]`
- (c) **铁律**：没有任何原文或外部来源支持的强断言必须在末尾标注 `[推断]`
- (d) 已标注 `📖` 来源的正文视为隐含 `[原文]`，可省略标签

### 10. 首次出现缩写解释检查

扫描本节中所有大写缩写（2-6 个大写字母），检查首次出现时是否附带一句解释（全称 + 基本含义）。对于领域内常见通用缩写（AI、ML、NLP、CPU、GPU），若上下文已清晰则豁免。

### 11. 公式格式全局扫描

**全文件扫描**（不按模块/区域区分，扫描整节课文件），逐项验证：

#### 11a. 反引号/代码块内公式检测
扫描所有反引号包裹内容（\`` `code` \`` 和 \```` ```code `````\``），若包含下列数学 Unicode 符号任意一个 → 告警并修正：`μ σ ∑ α β γ θ λ π ∈ ∉ ⊂ ⊆ ∩ ∪ ∀ ∃ ⊥ ∇ ∂ ≈ ∼ ≠ ≡ ≤ ≥ ∞`

#### 11b. Heading 公式缺失检测
扫描所有标题行（以 `#` 开头的行），检测是否包含上列数学 Unicode 符号但未被 `$` 包裹 → 告警，用 `$符号$` 包裹

#### 11c. 单元公式裸露检测
扫描全文件，查找非 `$` 包裹的 `μ`、`σ`、`∑` 等高频数学符号，若出现于正文（非代码块、非反引号）但无 `$` 包裹 → 告警，修正为 `$符号$`

**优先级**：三项检测在完整课程落盘后逐文件执行，不通过则修正后重检，不阻塞管线。修正不通过则标注 `> [!WARNING] 公式格式偏差` 在文件头部。

### 12. 节首术语集中度检查

扫描每节课程文件**前 30% 行**范围内：
- 若出现含 ≥3 个 `[[概念名]]` 双链的表格行（以 `|` 开头的行）且内容中无超过 5 个汉字的自然语言解释段 → 告警，要求 agent 将表格拆散为内联解释
- 若出现以 bullet 列表形式集中列出全部术语（连续 ≥3 个 bullet 行均含 `[[概念名]]` 或加粗术语）且无完整句子解释 → 告警，要求改为分散嵌入

**整改要求**：将集中术语表拆散，每个术语移动到它在正文中首次出现的位置，以 `[[概念名]]`（核心概念）或加粗（前置术语）或自然语言（背景术语）格式内联解释。

## 检测命令

```bash
# 块配对
for f in papers/*/course/*.md; do
  dollars=$(grep -o '\$' "$f" | wc -l)
  fences=$(grep -o '```' "$f" | wc -l)
  echo "$f: dollars=$dollars fences=$fences"
done

# 末字符检查
python3 -c "
import glob
for f in glob.glob('papers/*/course/*.md'):
    with open(f) as fh:
        text = fh.read()
    tail = text[-200:] if len(text) > 200 else text
    last_char = text[-1] if text else ''
    has_complete = any(c in tail for c in ['.', '。', '!', '！', '?', '？', ';', '；', '…'])
    in_middle = last_char in [',', '，', ':', '：', ';', '；', '、', '(', '（']
    print(f'{f}: complete_sent={has_complete} mid_char={in_middle}')
"

# 缩写检查
for f in papers/*/course/*.md; do
  grep -oP '\b[A-Z]{2,6}\b' "$f" | sort -u
done

# 11a. 反引号内公式检测
for f in papers/*/course/*.md; do
  # 提取所有反引号包裹内容，检查是否含数学 Unicode
  backtick_content=$(grep -oP '`[^`]+`' "$f" | grep -oP '[\x{03BC}\x{03C3}\x{2211}\x{03B1}\x{03B2}\x{03B3}\x{03B8}\x{03BB}\x{03C0}\x{2208}\x{2209}\x{2282}\x{2286}\x{2229}\x{222A}\x{2200}\x{2203}\x{22A5}\x{2207}\x{2202}\x{2248}\x{223C}\x{2260}\x{2261}\x{2264}\x{2265}\x{221E}]')
  if [ -n "$backtick_content" ]; then
    echo "⚠️  $f: 反引号内检测到数学符号"
    echo "   符号: $backtick_content"
  fi
done

# 11b. Heading 公式缺失检测
for f in papers/*/course/*.md; do
  # 查找 heading 行中数学符号但无 $ 包围
  grep -n '^#' "$f" | while IFS=: read -r line_num content; do
    # 检查是否包含数学符号但无 $
    if echo "$content" | grep -qP '[\x{03BC}\x{03C3}\x{2211}]' && ! echo "$content" | grep -q '\$'; then
      echo "⚠️  $f:$line_num heading 含数学符号但无 \$ 包裹: $content"
    fi
  done
done

# 11c. 公式裸露检测
for f in papers/*/course/*.md; do
  # 扫描非代码块、非反引号区域的裸露数学符号
  grep -nP 'μ|σ|∑|α|β|γ|θ|λ|π' "$f" | while IFS=: read -r line_num content; do
    # 跳过反引号行和代码块行
    if ! echo "$content" | grep -qP '^```|`[^`]*`'; then
      if ! echo "$content" | grep -q '\$'; then
        echo "⚠️  $f:$line_num 裸露数学符号: $content"
      fi
    fi
  done
done
```
