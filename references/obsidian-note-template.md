# Obsidian 笔记模板

本文件包含板块 A 的全部笔记产物模板。SKILL.md 负责编排，本文件负责"长什么样"。

所有笔记都要遵守两条铁律：
1. **每条总结标注【页码 / 章节名】**，无法定位的陈述不得冒充原文内容
2. **公式用 LaTeX 重现并解释符号**，不能只贴公式不说含义

**标注简化规则**（避免逐行重复标注）：
- **含内容才标**：标注放在真正含内容的末级标题上，纯分组标题（下面直接跟子标题而非内容）不标注
- **上级标过不重复**：范围性标注在其下所有内容默认适用，不再逐条标注
- **例外才单标**：表格、脚注、跨章节的内容单独标注出处

---

## 一、文献总结笔记（三种类型选一）

先判定类型（见 SKILL.md 步骤 1），再选对应模板。所有模板共用 frontmatter 和"一句话概述"开头。

### 类型 1：研究论文 —— 学术标准结构

```markdown
---
title: "<文献标题>"
authors: [<作者列表>]
year: <发表年份>
type: research
source: "<原文件名或 URL>"
read_date: <YYYY-MM-DD>
tags: []
---

# <文献标题> HMR

## 📌 一句话概述
<1-2 句说清这篇做了什么、解决了什么问题>

## 1. 背景与动机
- <要点>【第 N 页 / 引言】
- <要点>【第 N 页 / Sec.X】

## 2. 问题定义
- 形式化的问题陈述：<说明>【第 N 页 / Sec.2】
- 输入输出约定：<说明>【第 N 页 / Sec.2】

## 3. 方法（含公式）
- 核心思路：<说明>【第 N 页 / Sec.3.1】
- 关键公式：
  $$ <LaTeX 公式> $$
  - 符号说明：$x$ 表示...，$\theta$ 表示...
  - 直觉解释：<用自然语言讲清这个公式在干什么>

## 4. 实验
- 数据集：<说明>【第 N 页 / Sec.4】
- 基线方法：<说明>【第 N 页 / Sec.4】
- 关键结果：<说明>【第 N 页 / 表X】

## 5. 结论与局限
- 主要贡献：<说明>【第 N 页 / Sec.5】
- 局限性：<说明>【第 N 页 / Sec.5】

## 6. 相关工作
- <相关工作脉络>【第 N 页 / Sec.2 或 Sec.6】

## 🔗 术语
- [[术语1]]、[[术语2]]、[[术语3]]   ← 这些是拆分原子笔记的候选
```

> 术语只标注本篇文献**引入或重点解释**的概念。直接引用但未在本文中展开解释的已有术语不收入。

> 非典型研究论文（案例研究/技术报告/观点论文）归入此模板，把"## 4. 实验"标题按实际改为"## 4. 案例发现"/"## 4. 系统评估"/"## 4. 论据"等。

## 二、术语原子笔记（拆分阶段，用户主导）

用户拆分 `[[术语]]` 时，每个术语生成一个独立文件。文件名：`<术语名>.md`。

```markdown
# <术语名>

## 定义
<1-2 句精炼定义>

## 出处
- [[<文献标题> HMR]] 第 N 页 Sec.X

## 相关
- [[相关概念1]]、[[相关概念2]]
```

> 拆分由用户亲自操作，agent 只提供方法不代写。用户坚持要求代劳时，agent 需先说明"这样学习效果会差很多"，确认后才执行。

---

## 三、待学习原子笔记

目录名固定为英文：`to-learn/`。用户提出文献未解释的概念时生成。格式精简为三要素——概念名 + 出处 + 一句话解释：

```markdown
# 待学习：<概念名>

- 出自 [[<文献标题> HMR]] 第 N 页 Sec.X
- 💡 <一句话解释>
```

生成后向用户提供三项可选项（见 SKILL.md 待学习机制），由用户选择，不强制推送。

## 四、自测模块（默认关闭）

用户**明确要求开启**时，在文献总结笔记**末尾**单独添加一个「自测」模块。不对正文做任何修改——在原笔记内容之后追加，以问答形式呈现，让用户填写理解后交由 agent 评审。

模块格式：

```markdown
## ✅ 自测

请填写你的理解，填完后交由 agent 评审。

### Q1 · Sec X.X <考察点简述>

<问题>

____<在此填写你的理解>
```

- 考察点数量和位置由 agent 根据文献重点判断，聚焦在"理解了就真的懂了"的关键点上，通常 3-5 个问题
- 每个问题标注对应章节，方便用户在笔记中回溯查阅
- 问题以考察理解为主（"解释为什么""说明关系""对比异同"），不出数据记忆型题目

**评审规则（严格，不得奉承）**：
- 准确 → 回复"准确"，不做任何改动
- 不准确 → 指出哪里偏、为何偏，让用户重新填，**不要直接给答案**
- 用户明确说"直接告诉我答案" → 才给出答案
