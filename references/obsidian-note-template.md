# Obsidian 笔记模板

本文件包含板块 A 的全部笔记产物模板。SKILL.md 负责编排，本文件负责"长什么样"。

所有笔记产出到 vault 的 `HelpMeRead/` 子目录下（路径约定见 SKILL.md"命名约定"）。frontmatter 字段详解见 `references/frontmatter-schema.md`，此处只展示模板结构。

所有笔记都要遵守两条铁律：
1. **每条总结标注【页码 / 章节名】**，无法定位的陈述不得冒充原文内容
2. **公式用 MathJax 重现并解释符号**，不能只贴公式不说含义

**标注简化规则**（避免逐行重复标注）：
- **含内容才标**：标注放在真正含内容的末级标题上，纯分组标题（下面直接跟子标题而非内容）不标注
- **上级标过不重复**：范围性标注在其下所有内容默认适用，不再逐条标注
- **例外才单标**：表格、脚注、跨章节的内容单独标注出处

## 一、文献总结笔记（三种类型选一）

先判定类型（见 SKILL.md 步骤 1），再选对应模板。

### 类型 1：研究论文 —— 学术标准结构

```markdown
---

aliases:
  - <常用简称1>
  - <常用简称2>
authors:
  - <作者1>
  - <作者2>
year: <发表年份>
venue: <发表场所>
area:
  - <领域1>
  - <领域2>
type: research
source: "<原文 URL 或本地路径>"
pdf: "[[<PDF文件名>.pdf]]"
status: learning
read_date: <YYYY-MM-DD>
progress: 0
last_section: ""
sections: <课程总节数>
tags:
  - paper/<关键词>
up:
  - "[[HelpMeRead MOC]]"
related: []
---

## 📌 一句话概述

- <要点>【第 N 页 / 引言】
- <要点>【第 N 页 / Sec.X】

## 2. 问题定义

## 3. 方法（含公式）
- 核心思路：<说明>【第 N 页 / Sec.3.1】
- 关键公式：
  $$ <MathJax 公式> $$

```

> 术语只标注本篇文献**引入或重点解释**的概念。直接引用但未在本文中展开解释的已有术语不收入。

## 二、术语原子笔记（脚手架式，定义留空）

agent 在拆分阶段为用户生成骨架原子笔记到 `concepts/<术语名>.md`（去重后路径可能为 `papers/<slug>/<术语名>.md`，取决于用户在去重提示中的选择）。

```markdown
---
title: "<术语名>"
aliases:
  - <中文别名>
  - <其他常用名>
type: concept
area:
  - <所属领域>
defined_in:
  - "[[<当前论文> HMR]]"
up:
  - "[[<上位概念>]]"               ← 若存在
tags:
  - concept/<关键词>
---

# <术语名>

## 定义
> 你的理解：______（用自己的话说）

## 出处
- [[<当前论文> HMR]] 第 N 页 Sec.X

## 相关
- [[相关概念1]]、[[相关概念2]]
```

**跨论文去重时**：
- 若 `concepts/<术语名>.md` 已存在 → 提示用户选择：追加出处 / 新建独立文件 / 跳过
- 选"追加出处"→ 在已有文件的 `defined_in` 列表中添加 `[[当前论文 HMR]]`，`## 出处` 追加一行
- 选"新建独立文件"→ 路径为 `papers/<当前论文简称>/<术语名>.md`
- 选"跳过"→ 不生成本术语

> 拆分由 agent 用脚手架 + 聊天逐个引导的方式完成。用户**只动脑写定义（一两句），不动手建文件**。

## 三、待学习笔记（含生命周期）

目录：`to-learn/<概念名>.md`。agent 在用户遇到文献未解释的概念时自动生成（用户后续明确说"不要加入"才跳过）。

```markdown
---
title: "<概念名>"
type: to-learn
from:
  - "[[<文献标题> HMR]]"
raised_date: <YYYY-MM-DD>
status: open
resolved_to: []
tags:
  - to-learn
---

# 待学习：<概念名>

- 出自 [[<文献标题> HMR]] 第 N 页 Sec.X
- 💡 <一句话解释>
```

**生命周期**：
- `status: open` —— 刚开始，待学习
- `status: exploring` —— 正在查资料
- `status: resolved` —— 搞懂了，转成正式原子笔记（`resolved_to` 指向 `concepts/` 下的笔记），原 to-learn 文件保留作为学习轨迹

## 四、学习补充层（课程学完后追加到主笔记）

课程全部学完后，agent **只增不改**，在主笔记（HMR-<slug>.md）末尾追加以下章节。首次基于原文的笔记作为"原始记录"锁定（已经过页码校验+引用回查），学习补充层的内容带来源标签以区分可靠性。

```markdown
## 📝 学习补充

> 以下为课程学习过程中的反馈，**已与原文层分离**。

### 💡 深挖与澄清
<用户"这节没懂"触发的深挖内容，标注对应课程节>
- 【第 2 节 · 自注意力机制】<内容> [背景]
- ...

### ❓ 学习中的疑问
<将学习中提出的核心概念追问整理成待学习项或已解决记录>
- [[待学习概念A]]（已加入 to-learn）
- [[概念B]]（已解决）<内容> [推断]

### 🔗 跨论文联系
<agent 发现的与其他已读论文的关联>

```
