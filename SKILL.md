# Help Me Read —— 文献阅读辅助

本 skill 把一篇文献变成两类产物：**可学习的课程**（独立 HTML，不依赖 Obsidian）和 **Obsidian 知识库笔记**。执行时严格遵循下面的编排顺序，不要跳步、不要倒序。

## 触发与子命令路由

识别用户意图后按下表分发执行范围。无法判断时默认走全流程。

| 用户表达 | 执行范围 |
|---|---|
| 无关键词 / "完整" / "全套" / "帮我把这篇吃透" | 全流程 |
| "做成课程" / "教我" / "我想学懂这篇" | 仅板块 B（课程） |
| "总结" / "记到 obsidian" / "做个笔记" | 仅板块 A（笔记） |
| 直接提问（问号、"为什么""怎么理解"） | 仅板块 C（问答） |

> **子命令的步骤处理**：用户用子命令指定局部执行时，未被选中的板块及其依赖步骤直接略过，不要硬走。例如"只做课程"则跳过笔记写入和原子笔记拆分（拆分依赖笔记存在）；"只总结"则跳过课程生成与逐节引导。问答始终可被用户随时触发，不受子命令限制。

## 全流程编排

```
1. 获取原文 → 判定类型
2. 并行生成课程与笔记（两者都完成后才继续）
3. 课程开始前提醒"待学习"机制
4. 逐节引导学习（代为打开 HTML，用户反馈后开下一节）
5. 引导用户自行拆分原子笔记（只引导不代劳）
6. 开启问答
```

> 注意：问答可被用户随时打断式触发，不阻塞主流程；课程学完后主动告知"现在可以问我任何问题"。

## 步骤 1：获取原文与类型判定

**读取**（按输入类型）：
- 本地 PDF：用 Read 工具直接读取
- 论文 URL / 网页：用 `mcp__web_reader__webReader` 抓取
- **仅有论文标题（无 URL / PDF）**：用 WebSearch 搜索标题，找到候选结果后**向用户确认**是这篇再继续。确认格式："我找到了以下匹配，请问是这篇吗？（1）[标题] [链接]（2）[标题] [链接]"。若搜不到，提示用户提供 PDF 路径或 URL。确认后按 URL 模式继续。
- **URL 类型识别与转换**：抓取前先判断 URL 是否为摘要页（只有 metadata，无正文），是则转全文页再抓。常见转换：
  - `arxiv.org/abs/<id>` → `arxiv.org/html/<id>`（HTML 全文版优先，抓取更可靠；无 HTML 版再转 `arxiv.org/pdf/<id>`）
  - `arxiv.org/pdf/<id>` 已是全文，直接抓
  - DOI 链接（`doi.org/...`）→ 解析到出版社全文页；遇付费墙则提示用户提供可访问的 PDF
  - 期刊/会议详情页 → 找页面内的 "PDF" / "Full Text" 链接
  - 博客/文档站点 → 直接抓
- **抓取后校验**：若返回内容明显过短（只有摘要 + 作者信息，无正文段落），判定为抓到了摘要页，按上述规则转全文页重抓；仍失败则向用户索要 PDF
- 读取时记录可用的定位信息：PDF 页码、网页章节锚点/标题——后续笔记和问答都要引用

**类型判定**（决定走哪套模板，这是分流的关键）：

| 类型 | 判定特征 | 走哪套 |
|---|---|---|
| **研究论文** | 有原创方法 + 可复现实验 | 学术标准结构 |
| **综述** | 标题含 survey/review/综述；以分类体系为主线；无原创实验 | 领域地图结构 |
| **数据集论文** | 核心是发布一个数据集；标题常含 dataset/corpus/benchmark；重心是数据构成与标注 | 数据生命周期结构 |
| **其他**（案例研究/技术报告/观点论文等） | 不符合上述 | 归入研究论文模板，"实验"章节标题按实际内容灵活改为"评估/案例发现/论据" |

**判定原则**：**默认默默判定并告知结果**（如"我判定这是研究论文，走学术标准结构"），让用户有机会纠正但不必每次都问。只有当文献同时符合多类特征、或特征都很弱无法判断时，才主动向用户确认。

类型与对应模板、课程形态的细则，读取前先看：
- `references/obsidian-note-template.md`（三类笔记模板 + 原子笔记 + 填空位 + 待学习）
- `references/course-design-guide.md`（三类课程设计指南）

## 步骤 2：并行生成课程与笔记

本步骤同时产出课程 HTML 和 Obsidian 笔记，两者都完成后才进入下一步。

### 2.1 生成课程（板块 B）

**生成阶段**（一次性完成）：
1. 通读原文，识别核心概念链 / 流派全景 / 数据生命周期阶段（按类型）
2. 按学习路径重组为 **3-5 节**，每节聚焦 1-2 个核心概念
3. 每节顶部标注**可跳过条件**（"若你已熟悉 X，可跳过本节"）
4. **先展示课程大纲**（各节标题 + 可跳过条件），告知用户后**立即开始生成全部 HTML**——不主动等待确认，用户若不满意可随时打断重做
5. **选择主题模板**：默认使用浅色模板 `references/course-template.html`。每次生成前读取配置文件 `~/.help-me-read.json` 的 `theme` 字段——若为 `"dark"` 则改用 `references/course-template-dark.html`。首次运行默认 `theme: "light"`；用户说"用深色"/"切深色主题"时切换并回写配置。
6. 生成 HTML 文件到 `courses/<文献简称>/`，命名 `01-<概念名>.html`、`02-<概念名>.html`……
6. HTML 模板和设计细则见 `references/course-template.html` 和 `references/course-design-guide.md`

**单课五模块**（顺序固定）：
1. 学习目标 + 动机（承接上文）
2. 核心讲解 + 类比（日常类比讲直觉 → 形式化定义；公式用 KaTeX 渲染并配推导）
3. **(可选) 💻 代码实现**——当重要概念有可信源码时追加，可多次出现。详见 `references/course-design-guide.md`
4. 自测小问题（按难度浮动 1-3 题；答案用 `<details>` 折叠）
5. 小结 + 预告下节

### 2.2 写入 Obsidian 笔记（板块 A）

课程生成完毕后，紧接着写入笔记。

**确定路径**（按优先级，命中即停）：
1. 读取配置文件 `~/.help-me-read.json`，若有 `obsidian_vault` 字段则直接使用
2. 配置文件不存在或无该字段 → 向用户询问 Obsidian vault 的绝对路径 → 写入 `~/.help-me-read.json`，下次启动自动使用
3. 配置文件格式：
   ```json
   {
     "obsidian_vault": "D:\\Path\\To\\Vault"
   }
   ```
   注：Windows 路径中的反斜杠按 JSON 要求写成 `\\`

**文件名**：`<文献标题> HMR.md`，标题按下述规则清洗后作为文件名：
- 移除文件名非法字符：`: / \ ? * " < > |`
- 连续空格压缩为单个空格，去除首尾空格
- 示例：标题 "BERT: Pre-training Deep..." → `BERT Pre-training Deep... HMR.md`

**生成内容**：按类型选用对应模板（见 `references/obsidian-note-template.md`）：
- 研究论文：学术标准结构
- 综述：领域地图结构
- 数据集论文：数据生命周期结构

每条总结标注【页码 / 章节】。公式用 LaTeX 重现并解释符号。术语用 `[[术语名]]` 标注（作为后续拆分原子笔记的候选）。

写完后告知用户文件位置，并简要说明笔记里标注的术语是原子笔记候选。

## 步骤 3：课程开始前提醒待学习

课程和笔记均已生成完毕。列出课程大纲（各节标题 + 可跳过条件），然后**用独立醒目区块主动提醒待学习机制**：

> 💡 **待学习提醒**：学习中如果遇到文献里没解释清楚的概念或术语（如本文中直接使用的 RNN、GRU、beam search 等），随时告诉我——我会帮你存到 Obsidian 的 `to-learn/` 目录，并协助找相关资料帮你搞懂。

---

## 步骤 4：逐节引导学习

**逐节引导**：
- 用 `start "" "<第N节绝对路径>.html"` 代为打开（Windows 调系统默认浏览器）
- 告知用户："学完后告诉我，我帮你打开下一节；有不懂的地方随时可以问我。"
- 用户反馈"下一节/学完了" → 打开下一节
- 用户中途提问 → 切入问答（见步骤 6 标准），答完询问"继续学习吗"
- 最后一节学完 → 进入步骤 5（引导拆分原子笔记），并告知"现在可以问我关于这篇文献的任何问题"

---

## 步骤 5：引导拆分原子笔记（用户主导，不代劳）

**默认行为**：引导用户自己把笔记里的 `[[术语]]` 逐个展开成独立原子笔记。只给方法、不代写。理由：亲手拆分是知识内化的关键环节，代劳会让学习效果大打折扣。

**原子笔记规则**：每个术语一个文件 `术语名.md`，结构见 `references/obsidian-note-template.md`，含定义、出处（反向链接到文献笔记）、相关链接。

**用户坚持要代劳时**：先明确告知"这样学习效果会差很多，建议你自己拆"，得到用户确认后才代为执行。

---

## 步骤 6：问答（板块 C）

**首要原则（必须遵守）**：
> 力求一语中的，不绕弯、不把简单问题复杂化、不自行扩展。问什么答什么。能引用原文的尽量先引用原文，再补少量必要解释。

**完整标准**见 `references/qa-standards.md`，核心是四标签 + 强制溯源。每次回答后询问"还有问题吗 / 继续学习吗"。

**问答记录**：作为可选项，向用户询问"是否保存本次问答记录"后，单独存到 `qa/<文献简称>-<日期>.md`，**不**塞进总结笔记。

---

## 待学习机制（随时可用，不限时机）

用户提出文献里未解释的概念/术语时：
1. 存为原子笔记到 `to-learn/<概念名>.md`（目录名用英文），格式精简为三要素：概念名 + 出处 + 一句话解释（见 `references/obsidian-note-template.md`）
2. 向用户提供**可选项**协助学习：
   - 「搜索相关论文」：用 WebSearch 找该概念的综述/经典论文
   - 「找讲解资源」：搜索教程、博客、视频
   - 「先给一句话解释」：给简短解释帮用户判断是否值得深挖
3. 由用户选择，不强制推送全部

---

## 自测模块（默认关闭）

用户**明确要求开启**时才启用。在笔记末尾单独添加一个「自测」模块，以问答形式列出关键问题，留出答题位置让用户填写理解。用户填完后交由 agent 评审。

**评审规则（严格，不得奉承）**：
- 准确 → 回复"准确"，不做改动
- 不准确 → 指出哪里偏、为何偏，让用户重新填
- 用户明确说"直接告诉我答案" → 才给出答案

---

## 执行约束

- **顺序**：严格按全流程编排执行，不跳步、不倒序（除非用户用子命令指定局部）
- **定位**：所有总结和回答都要标注【页码/章节】，无法溯源的陈述不得冒充"原文说了"
- **简洁**：问答尤其要一语中的；总结也要精炼，不要为凑结构注水
- **不代劳**：拆分原子笔记由用户主导；自测评审要严格不将就
- **路径**：Obsidian 路径从环境变量读，没有就问；课程存项目内 `courses/`

---

## 命名约定（全文统一遵循）

避免每次执行路径/文件名不一致，统一如下：

| 对象 | 规则 | 示例 |
|---|---|---|
| **文献简称** | 文献标题转 kebab-case 英文小写，取关键词 ≤6 个词；中文标题取英文翻译或常用英文名 | "Attention Is All You Need" → `attention-is-all-you-need` |
| **课程子目录** | `courses/<文献简称>/` | `courses/attention-is-all-you-need/` |
| **课程文件** | `<两位序号>-<概念英文名>.html` | `01-why-attention.html` |
| **文献总结笔记** | `<标题清洗后> HMR.md`（标题移除非法字符 `: / \ ? * " < > \|`，空格压缩） | `Attention Is All You Need HMR.md` |
| **问答记录** | `qa/<文献简称>-<YYYY-MM-DD>.md` | `qa/attention-is-all-you-need-2026-06-21.md` |
| **待学习笔记** | `to-learn/<概念名>.md`（目录名固定英文） | `to-learn/positional-encoding.md` |
| **术语原子笔记** | `<术语名>.md` | `self-attention.md` |

文献简称一旦在某次会话中确定，本次会话内所有路径（课程目录、问答记录）都用同一个，保持一致。
