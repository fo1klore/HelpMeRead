# Help Me Read —— 文献阅读辅助

本 skill 把一篇文献变成两类产物：**可学习的课程**（Markdown，使用 Obsidian 兼容语法：callout/MathJax/双链）和 **Obsidian 结构化笔记**。两者**并行启动**，课程一次性全量生成、笔记同步一次性完成。课程学完后，笔记追加一层"学习补充"（只增不改，保护原文层可靠性）。

执行时严格遵循下面的编排顺序，不要跳步、不要倒序。

## 触发与子命令路由

识别用户意图后按下表分发执行范围。无法判断时默认走全流程。

| 用户表达 | 执行范围 |
|---|---|
| 无关键词 / "完整" / "全套" / "帮我把这篇吃透" | 全流程 |
| "做成课程" / "教我" / "我想学懂这篇" | 仅板块 B（课程） |
| "总结" / "记到 obsidian" / "做个笔记" | 仅板块 A（笔记） |
| 直接提问（问号、"为什么""怎么理解"） | 仅板块 C（问答） |
| "重做第 N 节" / "第 N 节重讲" | 重做该节课程（仅板块 B 局部） |
| "深挖第 N 节" / "第 N 节展开" | 在该节基础上展开更细（仅板块 B 局部） |

**歧义词处理**："总结""读一下""看一下"等既可能指完整学习也可能只要摘要的词，**一句话确认**："你是要 ① 完整学习（课程+笔记）还是 ② 只要笔记摘要？" 不要猜。

> **子命令的步骤处理**：用户用子命令指定局部执行时，未被选中的板块及其依赖步骤直接略过。例如"只做课程"则跳过笔记写入和原子笔记拆分；"只总结"则跳过课程生成与逐节引导。问答始终可被用户随时触发，不受子命令限制。

## 全流程编排

```
0. 首次运行引导（仅问 vault 路径，其余默认）
1. 获取原文 → 判定类型
2. 并行启动：课程一次性全量生成 + 笔记一次性完整生成
3. 课程开始前提醒"待学习"机制 + 列大纲
4. 开始学习（代为打开第一节，用户通过底部导航自主推进；进度按需回写 frontmatter）
5. 课程学完 → 引导拆分原子笔记（脚手架 + 聊天逐个引导）
```

> **贯穿全局的独立模块**（不占步骤号，随主线流程自动运行或随时触发）：
> - **学习补充层**：把学习/拆分过程中的反馈持续回写到主笔记 `## 学习补充`，与原文层物理分离
> - **待学习机制**：遇到未解释的概念自动存到 `to-learn/`，提供搜索协助和毕业机制
> - **跨论文联系**：发现与已读论文的关联时主动提示
> - **问答**：随时可打断式触发，不阻塞主流程。基于验证标签溯源回答。

## 步骤 0：首次运行引导（极简）

进行静默配置检查（内部完成，不向用户输出开始检查信号、检查过程或结果）。

若配置文件 `~/.help-me-read.json`（Windows 上 `~` 当 `%USERPROFILE%` 展开）**不存在**（首次使用），按以下顺序引导：

0. **最佳体验建议**（仅首次，之后不再重复）：
   > 💡 **最佳体验建议**：推荐在 Obsidian 社区插件中安装 **Claudian 插件**（[插件 GitHub 主页](https://github.com/YishenTu/claudian) | [中文安装教程](https://developer.aliyun.com/article/1712715)），然后将本 skill 安装到内置 agent。这样你可以在 Obsidian 单一窗口内完成「阅读 → 生成课程 → 学习 → 提问」全流程，无需来回切换。
   >
   > 是否继续当前配置？（回复"继续"即可，之后不再重复提醒）
   
   用户回复"继续"后进入下一步；若用户询问更多细节，简要说明 Claudian 是 Obsidian 与 Claude 的集成插件，可安装 skill 并在 vault 内运行。

1. **Obsidian vault 路径**：询问"你的 Obsidian vault 的绝对路径是？"说明"所有 Help Me Read 产物会集中存到 vault 的 HelpMeRead/ 子目录，方便和你的笔记互通，也能整体删除。"

2. **外部概念目录**（可选，可跳过）：
   询问："你是否有自己的存放原子概念的目录？比如你自己整理的 `ML/concepts/` 或 `consolidate/`。我会在后续去重时一并扫描这些目录中的笔记，避免重复创建已有概念。若没有，可暂时跳过，后续可随时重新设置。"
   
   - 用户提供路径（如 `ML/concepts`，多个用逗号分隔）→ 写入 `external_concepts_dirs`
   - 用户说"没有""跳过""暂时不设"→ 填入 `[]`

其余全部用默认值，写入配置文件后继续：

```json
{
  "obsidian_vault": "D:\\Path\\To\\Vault",
  "qa_record": false,
  "to_learn": true,
  "external_concepts_dirs": []
}
```

默认值含义（**不必在首次问用户**，让他们先看到第一份产物；想改随时可说）：
- `qa_record: false` —— 是否保存问答记录（开启则每次问答存为独立笔记）
- `to_learn: true` —— 是否开启待学习机制
- `external_concepts_dirs: []` —— 外部原子笔记目录列表，去重时一并扫描。设为空表示只扫描 HelpMeRead 自己的 `concepts/`

若配置文件**已存在且 `obsidian_vault` 路径有效**：跳过本步骤，不产生任何用户可见输出。

若配置文件存在但 `obsidian_vault` 路径不可访问：提示用户"你的 vault 路径（<路径>）似乎不可访问，是否需要更新？"

> **能力发现**：不把所有功能堆在首次引导里。每个流程节点结束时，根据当前状态输出简短的"接下来可以…"提示（按上下文，不固定大卡片）。详见各步骤末尾。

## 步骤 1：获取原文与类型判定

### 读取（按输入类型）

- **本地 PDF**：用 Read 工具直接读取
- **论文 URL / 网页**：用 `mcp__web_reader__webReader` 抓取
- **仅有论文标题（无 URL / PDF）**：用 WebSearch 搜索标题，找到候选结果后**向用户确认**是这篇再继续。确认格式："我找到了以下匹配，请问是这篇吗？（1）[标题] [链接]（2）[标题] [链接]"。若搜不到，提示用户提供 PDF 路径或 URL。确认后按 URL 模式继续。
- **URL 类型识别与转换**：抓取前先判断 URL 是否为摘要页（只有 metadata，无正文），是则转全文页再抓。常见转换：
  - `arxiv.org/abs/<id>` → `arxiv.org/html/<id>`（HTML 全文版优先，抓取更可靠；无 HTML 版再转 `arxiv.org/pdf/<id>`）
  - `arxiv.org/pdf/<id>` 已是全文，直接抓
  - DOI 链接（`doi.org/...`）→ 解析到出版社全文页；遇付费墙则提示用户提供可访问的 PDF
  - 期刊/会议详情页 → 找页面内的 "PDF" / "Full Text" 链接
  - 博客/文档站点 → 直接抓
- **抓取后校验**：若返回内容明显过短（只有摘要 + 作者信息，无正文段落），判定为抓到了摘要页，按上述规则转全文页重抓；仍失败则向用户索要 PDF

### 原文获取鲁棒性（只处理高频问题，其余兑底）

**PDF 两栏抽取检测**：Read 后若段落不连贯（句子中逤断开、左右栏交错）、公式乱码、图表消失 → 判定抽取失败。

**Fallback 链**：
1. Read 失败 → 尝试 arxiv HTML 版（若可获取，抽取最干净）
2. arxiv HTML 返回 404 或内容明显是错误页 → 回退 PDF 模式
3. 仍失败 → 提示用户"原文抽取遇到问题（两栏错乱/图表丢失），建议提供 HTML 版链接或更干净的 PDF"

**图片抽取**（论文原图是最高教学价值，必须提取。按优先级逐级尝试，自动降级，不卡流程）：

1. **arxiv HTML 原图**（优先）：若论文有 arxiv 版本，优先从 HTML 版抓取 `<img>` 原图（作者上传的原始 PNG/PDF，质量最佳）。用 WebFetch 或 webReader 获取。
2. **pdfimages 提取 PDF 内嵌图**（优先于 PyMuPDF）：poppler-utils 的 `pdfimages` 返回的 bbox 通常更准确，图与文本分离更干净。先检测 poppler-utils 是否可用：
   - 已安装 → 用 `pdfimages -png` 提取所有内嵌图；**禁用 pdftoppm**（整页栅格化）
   - 未安装 → 提示安装（macOS: `brew install poppler`，Linux: `apt install poppler-utils`，Windows: 需手动下载 poppler 二进制包并配 PATH）
     - 安装成功 → 继续提取
     - 安装失败或不可用 → 自动降级到路径 3
3. **PyMuPDF 提取 PDF 内嵌图**：pdfimages 不可用时走此路径。检测 PyMuPDF 是否可用：
   - 已安装 → 用 `page.get_images()` 提取 PDF 内嵌图对象。**渲染时使用图片自身的 bbox 作为 clip 矩形**（非页面 MediaBox），**禁用 `page.get_pixmap()` 做整页截图**。若内嵌图原始尺寸 ≤ 200px（宽或高），**不做 2x 放大**——直接原尺寸保存，避免过度放大导致内容丢失
   - 未安装 → 提示用户安装（`pip install PyMuPDF`）
     - 安装成功 → 继续提取
     - 安装失败 → 自动降级到路径 4
4. **矢量图回退**：当 `get_images()` 返回 0 张有效图时，用 `page.get_drawings()` 检测页面是否有矢量图形内容。若有，从原文文本中推断 Figure 所在位置（页码/区域），对该页面区域做**限定 clip 渲染**——clip 矩形严格限定为图/表所在区域，**严禁渲染整页**。矢量图渲染倍率 2x
5. **全部失败**：告知用户「自动提取失败，你可以手动截图保存到 `course/assets/` 目录，然后告诉我"重新嵌入图片"」。课程中对应位置标注「📎 参见原文 Figure N」——**严禁用 ASCII art 或文字描述臆造示意图**。

**图片质量控制**——提取后、落盘前执行以下过滤：
1. **尺寸下限过滤**：丢弃长或宽 < 100px 的图（图标、分隔线、装饰元素等噪音）
2. **尺寸上限过滤**：丢弃图宽 > 页面宽度 × 80% **且** 图高 > 页面高度 × 80% 的图（全页文字截图，不是图/表）。真正的论文图表很少超过半页
3. **纸张比例过滤**：若图宽高比接近 A4/US Letter 纸张比例（1.25–1.45）且尺寸 > 页面 60%，标记为可疑——让 agent 人工判断（可能是跨页大图，不自动丢弃）
4. **文字边缘裁切**：渲染后检查图片左右两侧是否有大块同色像素（文字栏特征），若有则向内收缩边界后重新裁切保存
5. **按需落盘**：只把确定要嵌入课程/笔记的图存入 `papers/<文献简称>/course/assets/`，其余当场丢弃
6. **噪音清理**：课程 + 笔记全部生成完毕后，扫描 `assets/`，删除未被任何文件 `![[...]]` 引用的残留图

每张成功提取的图标注 Figure 编号 + 页码，存入 `papers/<文献简称>/course/assets/figure-N.png`，课程中用 `![[assets/figure-N.png]]` 嵌入。表格不提取（极其困难），用文字描述 + 标注原文表号。

**边缘情况统一兑底**（不一一写死处理规则，一句话提示用户）：
- 扫描版 PDF（图片型，无文本层）→ 提示"这是扫描版，抽取不到文本，建议找原生 PDF 或 HTML 版"
- 付费墙 → 提示用户提供 PDF，可选建议"试试 unpaywall 或作者主页找 OA 版"
- 超长论文（50+ 页）→ 分批抓取/分节处理，告知用户"论文较长，分段处理"
- JS 渲染页面 → 提示"页面是动态渲染，抓取失败，建议提供 PDF"

读取时记录可用的定位信息：PDF 页码、网页章节锚点/标题——后续笔记和问答都要引用。

### 类型判定（决定走哪套模板，这是分流的关键）

| 类型 | 判定特征 | 走哪套 |
|---|---|---|
| **研究论文** | 有原创方法 + 可复现实验 | 学术标准结构 |
| **综述** | 标题含 survey/review/综述；以分类体系为主线；无原创实验 | 领域地图结构 |
| **数据集论文** | 核心是发布一个数据集；标题常含 dataset/corpus/benchmark；重心是数据构成与标注 | 数据生命周期结构 |
| **其他**（案例研究/技术报告/观点论文等） | 不符合上述 | 归入研究论文模板，"实验"章节标题按实际内容灵活改为"评估/案例发现/论据" |

**判定原则**：**默认默默判定并告知结果**（如"我判定这是研究论文，走学术标准结构"），让用户有机会纠正但不必每次都问。只有当文献同时符合多类特征、或特征都很弱无法判断时，才主动向用户确认。

类型与对应模板、课程形态的细则，读取前先看：
- `references/obsidian-note-template.md`（三类笔记模板 + 原子笔记脚手架 + 待学习 + 学习补充层）
- `references/frontmatter-schema.md`（四类笔记 + MOC 的 frontmatter 全字段）
- `references/course-design-guide.md`（三类课程设计指南）

## 步骤 2：并行启动（课程全量生成 + 笔记一次性完整）

本步骤**同时启动**两个产物。课程一次性全量生成（用户可随时打断）；笔记按论文原结构一次性完整生成（与课程独立，用户在学习过程中就能查阅）。两者任一先完成不阻塞另一个。

### 2.1 生成课程（板块 B）

**生成阶段**（一次性全量生成，可随时打断调整）：
1. 通读原文，识别核心概念链 / 流派全景 / 数据生命周期阶段（按类型）
2. 按学习路径重组为 **N 节**（节数见 `course-design-guide.md` 弹性规则：默认 3-5，最少 1，超 7 需向用户说明理由；综述单独放宽到 6-8）
3. 每节开头放折叠的**课程地图**（全部节标题列表，当前节高亮），然后标注**可跳过条件**（"若你已熟悉 X，可跳过本节"）
4. **先展示课程大纲**（各节标题 + 可跳过条件 + 总节数），告知用户后**立即一次性生成全部课程 Markdown 文件**——不主动等待确认，用户若不满意可随时打断，指定某节重做
5. **一次性生成全部课程**到 `papers/<文献简称>/course/`，命名 `01-<概念名>.md`、`02-<概念名>.md`……全部生成完毕后告知用户文件位置及总节数，用户即可开始学习
6. 课程设计细则、单课模块、callout 用法见 `references/course-design-guide.md`

**单课模块**（顺序固定）：
0. **课程地图**（每节开头，折叠 callout `> [!map]-`，列出全部节标题，当前节 `→` 加粗。详见 `course-design-guide.md`）
1. 学习目标 + 动机（承接上文）
2. 核心讲解 + 类比（日常类比讲直觉 → 形式化定义；公式用 MathJax `$...$` / `$$...$$` 渲染并配推导）
3. **(可选) 💻 代码实现**——当重要概念有可信源码时追加，可多次出现
4. 自测小问题（按难度浮动 1-3 题；**问题直接展示，答案用 callout 折叠**——只折答案不折问题）
5. 小结 + 预告下节
6. **(可选) 🔗 延伸**——精选 1-3 个高质量外部链接（3Blue1Brown、distill.pub、经典 lecture 等），只链出去不抓内容进来

### 2.2 写入 Obsidian 笔记（板块 A）

**与课程并行启动**，一次性完整生成。

**路径**：`<vault>/HelpMeRead/papers/<文献简称>/HMR-<文献简称>.md`（vault 路径从配置文件读）。

**文件名**：`HMR-<文献简称>.md`（以 HMR- 为前缀，含文献标识，避免与其他笔记撞名；标题放 frontmatter）。

**生成内容**：按类型选用对应模板（见 `references/obsidian-note-template.md`）：
- 研究论文：学术标准结构
- 综述：领域地图结构
- 数据集论文：数据生命周期结构

每条总结标注【页码 / 章节】，**带验证标签**（见步骤"内容准确性校验"）。公式用 MathJax 重现并解释符号。核心概念用 `[[概念名]]` 标注（作为后续拆分原子笔记的候选）。笔记的方法和实验章节中嵌入关键架构图/流程图/结果图（与课程共用 `course/assets/`，不额外提取）。

写完后告知用户文件位置，并简要说明笔记里标注的核心概念是原子笔记候选。

## 步骤 3：课程开始前提醒 + 列大纲

课程和笔记均已开始生成。列出课程大纲（各节标题 + 可跳过条件 + 总节数）。然后**用独立醒目区块主动说明待学习机制**：

> 💡 **待学习机制**：学习过程中如果遇到文献里没解释清楚的概念或术语，告诉我即可。我会先给你一句简短解释帮你判断是否值得深挖，然后自动存入 Obsidian 的 `to-learn/` 目录。之后你还可以选择让我搜索相关论文、找教程资源帮你彻底搞懂。

## 步骤 4：开始学习

课程全部生成后，底部导航（`← 上一节 | 下一节 →`）已嵌入各课程文件，用户可直接在 Obsidian 内点击导航链接自主推进，**无需每节向 agent 反馈**。

**代为打开第一节**：
- 用平台对应命令打开第一节课程文件（详见"跨平台"节）
- 告知用户："课程文件底部有导航链接，你可以直接在 Obsidian 里点击进入下一节。学习中遇到任何问题随时回来问我。"

**按需交互**（用户随时可用，不强制）：
- 用户中途提问 → 切入问答（见问答独立模块标准），答完询问"继续学习吗"
- 用户说"重做第 N 节" → 只重生那一节
- 用户说"深挖第 N 节" → 在那一节基础上展开更细
- 用户告知"学完了全部课程" → 进入步骤 5（引导拆分原子笔记）。学习补充层已在学习全程自动回写到主笔记，无需用户额外操作。并告知"现在可以问我关于这篇文献的任何问题"

**进度追踪**：
- 用户告知学完全部课程后，**回写 frontmatter**：`status: learned`
- 会话续接时，直接问用户当前学到哪节即可

---

## 步骤 5：引导拆分原子笔记（脚手架 + 聊天逐个引导）

**原则：要用户动脑（写定义），不要用户动手（建文件、抄出处）。**

**脚手架生成**：为笔记里每个 `[[核心概念]]` 生成骨架原子笔记到 `concepts/<概念名>.md`。

必选区块（每篇必须有）：
- frontmatter 全填好（含 `defined_in` 指向当前论文）
- `## 定义` 留空，写成 `> 你的理解：______（用自己的话说）`——让用户主动思考
- `## 出处` 自动填（反向链接到 HMR 笔记的具体页码/章节）
- `## 🔍 深入理解` agent 撰写深层解释（Obsidian 折叠块 `> [!help]-`，用户先自己思考再展开看）
- `## 相关` 自动填（从笔记里提取的相关核心概念）

可选区块（有则加，没有则跳过）：
- `## 📐 关键公式` 仅当原文有明确公式时添加（MathJax 格式 + 符号说明）
- `## 🏗️ 结构` 仅当有架构图/流程图时添加（嵌入图片 + 一句话说明）
- `## ⚖️ 分析` 仅当可分析优劣/边界/对比时添加
- `## 🔗 延伸探索` 仅当分析的「对比」涉及外部概念时添加（空链接骨架 + 一句话关系说明）

**跨论文去重**：生成前在以下位置搜索 `<概念名>.md`：
- `concepts/`（默认）
- 配置中 `external_concepts_dirs` 列出的所有目录
任一位置存在同名文件即判定为已存在，提示用户"这个概念你之前在 [[X 论文]] 拆过了，本次只追加 [[本论文]] 到 `defined_in`，不重写定义？（① 追加出处 ② 新建独立文件 ③ 跳过）"
- 用户选 ① → 追加到已有文件的 `defined_in` 列表，定义不动
- 用户选 ② → 新建 `papers/<本论文简称>/<概念名>.md` 独立文件
- 用户选 ③ → 跳过

**聊天逐个引导 + 纠错**（务必执行完整流程，不跳过）：
- agent 在对话里逐个抛出："接下来拆 self-attention。先看原文怎么说的：[引用原文定义]。用你自己的话说说 self-attention 是什么？"
- 用户口述后，agent 执行以下纠错流程（不直接润色填入）：

  1. **评价**：将用户理解与原文定义做语义对比，指出遗漏的关键点和错误点（若有）。措辞用「补充」而非「纠正」——如"你说的基本抓住了核心，不过原文还强调了一点 X……"；明显错误时引用原文「原文的表述是 [引用]，和你说的相比差别在 Y……」。
  2. **反查课程**：检查对应的课程节中对该概念的讲解是否足够清晰、详细。
     - 若课程解释不足 → 先**就地增补课程对应节**的内容（补充遗漏要点、增加推导步骤、细化类比），然后引导用户重新查看增补后的课程内容；
     - 若课程已足够 → 引用课程中的关键段落，引导用户再读一遍。
  3. **拆解问题**：将复杂概念拆成 2-3 个子问题，一问一答，引导用户逐点理清思路。
  4. **引导修正**：引导用户补全遗漏点、修正偏差，用自己的话重新组织后再说一遍。
  5. **确认填入**：agent 润色用户修正后的定义，填入 `## 定义`。
  6. **回写学习补充层**：若本次拆分产生了纠错点、认知澄清或反查课程后的增补，**同步追加**到主笔记 `## 学习补充`，标注对应核心概念名（这是学习补充层独立模块在本阶段的持续回写，见文末贯穿模块说明）。

- 一个拆完继续下一个，显示进度"已完成 3/8"
- 用户随时可说"跳过这个""下次再拆"

**触发时机**：学完后批量引导 + 学习中用户说"拆这个"可随时拆单个。

> **接下来可以…**：拆完原子笔记后，告知用户：
>
> 🎉 这篇文献的完整学习流程已收官！你现在可以：
> - 💬 **继续向我提问**——这篇论文的任何细节、公式推导、设计动机
> - 🔍 **补充课程细节**——觉得某节讲得不够，告诉我重做或深挖
> - 📚 **发现相关论文**——我会扫描你已读的文献，帮你找跨论文的关联
> - 📖 **开始下一篇文献**——给我标题或链接，开启新的学习周期

---

## 待学习机制（随时可用，不限时机）

用户提出文献里未解释的概念/核心概念时，按以下流程处理：

1. **先给一句话解释**——用最简短的话帮用户判断这个概念是否值得深挖
2. **自动存入 to-learn**——不向用户询问"是否加入"，直接存为 `to-learn/<概念名>.md`（格式精简为三要素：概念名 + 出处 + 一句话解释；带生命周期 frontmatter）。仅在用户后续明确说"不要加入"时跳过
3. **提供可选项**协助学习：
   - 「搜索相关论文」：用 WebSearch 找该概念的综述/经典论文
   - 「找讲解资源」：搜索教程、博客、视频
4. 由用户选择，不强制推送全部

**毕业机制**：to-learn 笔记有生命周期（`status: open → exploring → resolved`）。用户搞懂后可转成 `concepts/` 下的正式原子笔记，原 to-learn 文件标 `resolved` 保留作为"我曾经不会"的记录。

---

## 学习补充层（贯穿全程持续回写）

本模块贯穿步骤 4（课程学习）和步骤 5（拆分原子笔记），**随主线流程自动回写**，不占步骤号、不是用户单独推进的阶段。拆出来的章节引用避免与步骤号混淆——它不叫"步骤 X"，是一个横切关注点。

**只增不改**——首次基于原文的笔记作为"原始记录"锁定（已经过页码范围校验+引用回查，是"已验证"的可信内容）；学习过程的反馈是主观的、未经验证的，混进原文层会污染可靠性。

**回写时机**：
- **课程学习阶段**（步骤 4）：学习中冒出的待学习概念、用户"这节没懂"触发的深挖 → **实时**追加到主笔记 `## 学习补充`（待学习概念另同步生成 `to-learn/` 笔记）
- **拆分原子笔记阶段**（步骤 5）：纠错、反查课程后的增补、认知偏差澄清 → **同步**追加到 `## 学习补充`，标注对应核心概念名

**追加内容类型**：
- 学习中冒出的待学习概念 → 追加到 `## 学习补充` 或生成 `to-learn/` 笔记
- 用户"这节没懂"触发的深挖 → 追加到 `## 学习补充`，标注对应课程节
- 拆分原子笔记时的纠错与认知澄清 → 追加到 `## 学习补充`，标注对应核心概念
- 跨论文联系建议（「跨论文联系」模块发现的）→ 追加到 frontmatter `related` 字段
- 关键问答的精华（可选，用户决定）→ 追加到 `## 学习补充`，标 `[背景]`/`[推断]` 等标签

学习层的追加内容自带来源标签，用户能分清哪些是基于原文、哪些是学习过程的产出。

## 跨论文联系（生成新笔记时主动建议）

生成新论文笔记时，agent 扫描已有笔记（看 `area` / `aliases` / `defined_in` / frontmatter），发现相关论文**主动提示**：

> 你之前读过 [[X 论文]]，这篇和它在 Y 上有关联。要我对比一下吗？

用户决定是否深入。对比内容可写入 `concepts/<topic>-comparison.md` 或追加到相关笔记的 `related` 字段。不自动生成（可能不准且费 token）。

## 问答（随时可打断式触发）

本模块贯穿全流程——从课程学习到拆分原子笔记，用户在任何阶段都可以随时提问，不阻塞主线。

**首要原则（必须遵守）**：
> 力求一语中的，不绕弯、不把简单问题复杂化、不自行扩展。问什么答什么。能引用原文的尽量先引用原文，再补少量必要解释。

**完整标准**见 `references/qa-standards.md`，核心是**带验证标签的四标签** + 强制溯源。每次回答后询问"还有问题吗 / 继续学习吗"。

**问答记录**：作为可选项，向用户询问"是否保存本次问答记录"后，单独存到 `papers/<文献简称>/qa-<日期>.md`，**不**塞进总结笔记。

---

## 内容准确性校验（三层 + 诚实措辞）

> **诚实声明**：本 skill 提供"**带验证标签的溯源**"，不 100% 保证可靠。所有 `[原文]` 标注经过校验后会带验证状态，用户能区分哪些可信。完全消除幻觉不可能。

生成笔记和回答时执行三层校验：

1. **页码范围校验**：已知论文总页数 N，所有 `【第 X 页】` 必须 `1 ≤ X ≤ N`，超出标记可疑
2. **引用回查**：生成后对每条 `[原文]` 标注，在抓取的原文文本里搜索关键词：
   - 找到 → 标 `[原文·已验证]`
   - 找不到 → 降级为 `[原文·未验证]`，或建议改成 `[推断]`
3. **标记制**：用户能区分"已验证/未验证"，知道哪些可信，留个心眼

**措辞**：所有对外文档（README、SKILL.md 自描述）用"带验证标签的溯源"，不用"强制溯源""绝对可靠"。

## 跨平台

- **代为打开文件**：优先使用 Obsidian URI scheme：`obsidian://open?vault=<vault目录名>&file=<HelpMeRead/下的相对路径>`。vault 目录名取配置中 `obsidian_vault` 路径的最后一级（如 `D:\Path\To\MyVault` → `MyVault`）。URI 打开失败时回退到平台命令（Windows `start ""`、macOS `open`、Linux `xdg-open`）
- **PDF 图片抽取**：优先 arxiv HTML 原图（零依赖）；PDF 路径按需提示安装 PyMuPDF（`pip install PyMuPDF`，用 `page.get_images()` 取内嵌图），poppler-utils（仅 `pdfimages`）作为兜底。**禁用整页栅格化工具（pdftoppm / `page.get_pixmap()`）做图片提取**。提取后按尺寸过滤 + 按需落盘 + 噪音清理（详见步骤 1）
- **配置文件**：`~/.help-me-read.json`，Windows 上 `~` 当 `%USERPROFILE%` 展开

## 执行约束

- **顺序**：严格按全流程编排执行，不跳步、不倒序（除非用户用子命令指定局部）
- **定位**：所有总结和回答都要标注【页码/章节】，**带验证标签**；无法溯源的陈述不得冒充"原文说了"
- **简洁**：问答尤其要一语中的；总结也要精炼，不要为凑结构注水
- **只增不改**：笔记原文层锁定，学习反馈追加到 `## 学习补充`
- **要动脑不要动手**：原子笔记拆分用脚手架 + 聊天引导，不代劳也不让用户自己建文件
- **诚实**：不宣称绝对可靠，用验证标签让用户自行判断

## 命名约定（全文统一遵循）

避免每次执行路径/文件名不一致，统一如下。所有产物在 vault 的 `HelpMeRead/` 子目录下：

| 对象 | 规则 | 示例 |
|---|---|---|
| **文献简称** | 文献标题转 kebab-case 英文小写，取关键词 ≤6 个词；中文标题取英文翻译或常用英文名 | "Attention Is All You Need" → `attention-is-all-you-need` |
| **文献总结笔记** | `papers/<文献简称>/HMR-<文献简称>.md`（HMR- 前缀 + 文献标识） | `papers/attention-is-all-you-need/HMR-attention-is-all-you-need.md` |
| **课程子目录** | `papers/<文献简称>/course/` | `papers/attention-is-all-you-need/course/` |
| **课程文件** | `<两位序号>-<概念英文名>.md` | `01-why-attention.md` |
| **课程图资源** | `papers/<文献简称>/course/assets/` | `assets/figure-1.png` |
| **问答记录** | `papers/<文献简称>/qa-<YYYY-MM-DD>.md` | `papers/attention-is-all-you-need/qa-2026-06-22.md` |
| **核心概念原子笔记** | `concepts/<概念名>.md`（跨论文共享，去重） | `concepts/self-attention.md` |
| **待学习笔记** | `to-learn/<概念名>.md`（目录名固定英文） | `to-learn/positional-encoding.md` |
| **MOC** | `HelpMeRead MOC.md`（固定，vault 的 HelpMeRead/ 根） | `HelpMeRead MOC.md` |

文献简称一旦在某次会话中确定，本次会话内所有路径（课程目录、问答记录）都用同一个，保持一致。

**重名处理**：生成简称后检查 vault 中 `papers/<简称>/` 是否已被占用：
- 目录不存在 → 直接使用
- 目录存在且为**同一篇论文**（已有 `HMR-<简称>.md` 的 `title` 相同）→ 提示用户"这篇论文已有产物"，给出选项：① 重新生成（覆盖旧产物，需二次确认）② 保留旧产物，取消本次
- 目录存在且为**不同论文**（`title` 不同）→ 告知用户冲突，自动追加消歧后缀：先试 `<简称>-<年份>`，仍冲突则追加作者姓（`<简称>-<年份>-<姓>`），全部失败则请用户手动指定简称
